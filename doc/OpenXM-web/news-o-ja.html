  • llvm 12 をもとに作られた Apple clang 13 
    clang --version
Apple clang version 13.0.0 (clang-1300.0.29.30)
    によりコンパイルされた M1 Mac 上の Risa/Asir はあるデータの元で dp_ptod が segmentation fault をおこします. この原因は optimizer がゴミでない領域を Boehm GC に回収させてしまうような コードを生成するためです (局所変数-8 がレジスターに入っているだけのコード. 局所変数は stack に格納されない.). この問題点の workaround は この commit で与えられていますが, 他の部分でも同じような問題が生じる可能性があるので, homebrew llvm@13 に付属する clang を用いて Risa/Asir をコンパイルすべきです. (2021-12-28)

    D-加群についての機能が充実しました.
    1.3.1-3 以前がインストールしてある場合, 下記のようにインストールして下さい. 
         sudo su
     apt-get install make nkf
     dpkg --purge openxm
     apt-get install openxm
    TeXmacs インタフェース, desktop アイコンをインストールするには 
        apt-get install openxm-skel
    apt-get install openxm-gnome-desktop
    です.
    apt-get を使えない時(つかってはいけないとき)は, まず openxm- ではじまるすべてのファイルを http://fe.math.kobe-u.ac.jp/KnoppixMath からダウンロードしたあと, 次のようにして手動でインストールします. 
       sudo su
   dpkg --install openxm_*.deb
   dpkg --install openxm-skel*.deb
    なお openxm_*.deb はネットワークインストーラで本体は internet からダウンロードされます.
